# VIVIM Frontend "AI Slop" Audit & Refinement Plan (v1.0)

## 1. Audit Summary: The "Sovereign" vs "Generic" Gap
The VIVIM frontend is visually stunning but suffers from **Aesthetic and Structural Slop**—the result of high-volume AI generation that prioritizes "looking like an AI app" over "functioning as a sovereign memory platform."

---

## 2. Identified "Slop" Categories

### 2.1. Visual Tropes (Aesthetic Slop)
*   **Color Over-Saturation:** Excessive use of `from-violet-500 to-cyan-500` gradients. This has become the "standard AI" aesthetic. 
*   **Boilerplate Visuals:** `HeroVisual.tsx` uses a generic "pulsing brain" SVG. This is a common trope that lacks the unique "ACU" (Atomic Chat Unit) identity.
*   **Glassmorphism Overkill:** Every component uses `glass-card`. It makes the UI feel "floaty" rather than "solid" or "sovereign."

### 2.2. Copy & Content Slop
*   **The "Stub" Problem:** Multiple UI elements (e.g., "Stub Context" badge in `chat/page.tsx`) are hardcoded placeholders. 
*   **Generic Messaging:** Phrasing like "Works with Any AI Provider" is repeated 4+ times on the landing page.
*   **Static Scorecards:** The `PROBLEMS` array in `page.tsx` is a massive (200+ line) hardcoded JSON object. This should be a separate data file or a CMS-driven resource.

### 2.3. Architectural Bloat (Code Slop)
*   **The Monster File:** `packages/frontend/src/app/page.tsx` is **1,480 lines long**. It mixes:
    *   Large data constants (Problems, Features, Tiers).
    *   Complex Framer Motion animations.
    *   Deeply nested Tailwind components.
    *   Modal/Dialog logic.
*   **Impact:** Extremely high cognitive load for maintenance; likely contains redundant "just-in-case" styles generated by the AI.

---

## 3. Refinement Blueprint: "De-Slopping" the UI

### 3.1. Visual Identity Pivot
*   **From "AI Glow" to "Knowledge Vault":** Introduce more "Industrial" or "Cryptographic" elements. Use sharper borders, mono-spaced fonts for data, and a more restricted, intent-driven color palette.
*   **ACU Visualization:** Replace the generic brain SVG with a **Grid of Atoms** visualization that actually reflects the user's `AtomicChatUnit` count.

### 3.2. Structural Refactoring
*   **Component Extraction:** Break `page.tsx` into:
    *   `src/components/landing/ProblemScorecard.tsx`
    *   `src/components/landing/RightsLayerTiers.tsx`
    *   `src/components/landing/HeroSection.tsx`
*   **Data Separation:** Move the `PROBLEMS` and `LAYER_DATA` arrays to `src/lib/constants/landing-data.ts`.

### 3.3. Identity Lifecycle Integration
*   **Ripening Visualization:** Replace the "Stub Context" badge with an **Identity Ripening Meter** that reflects the `VirtualUser.confidenceScore` (Stranger → Known).
*   **Dynamic Context Viz:** The `context-budget-viz.tsx` should be moved from the landing page into the *actual* chat interface, showing live token usage per layer (L0-L7).
